Aggregation Basics

GROUP BY collapses rows into aggregates per key.

🧒 ELI5

Count your toys

You count how many toys of each type you have.

🔗 ANALOGY

Classroom rollup

You group students by class and count them.

Technical

What it means in real code

Definition: GROUP BY groups rows by columns, then applies aggregates like COUNT, SUM, AVG.

Why it matters

  • Summaries
  • Analytics
  • Reporting

Common pitfall

  • Selecting non-grouped columns
  • Wrong aggregates
  • Missing filters

Basic group by
SELECT status, COUNT(*)

Mini takeaway

  • Be explicit with columns.
  • Use HAVING for group filters.

Section 2 of 4 · ~7 mins

HAVING vs WHERE

WHERE filters rows before grouping; HAVING filters groups after aggregation.

🧒 ELI5

Sort then count

You remove bad apples before counting the rest.

🔗 ANALOGY

Reject first, then tally

You exclude invalid entries before counting totals.

Technical

What it means in real code

Definition: WHERE applies before GROUP BY; HAVING applies after.

Why it matters

  • Correct results
  • Efficient filtering
  • Clear intent

Common pitfall

  • Using WHERE on aggregates
  • HAVING without GROUP BY
  • Incorrect totals
WHEREHAVINGOrder
Filter rows→Group→Aggregate→Filter groups

Interview answer

“I use WHERE for row filters and HAVING for aggregate filters.”

ClauseStage
WHEREBefore grouping
HAVINGAfter grouping
HAVING example
SELECT customer_id, COUNT(*)
FROM orders
WHERE status = 'PAID'
GROUP BY customer_id
HAVING COUNT(*) >= 3;
Warning

HAVING is slower than WHERE

Filter early when possible.

Mini takeaway
  • WHERE filters rows.
  • HAVING filters groups.
  • Use WHERE first.
Quiz

When does HAVING run?

Correct answer: B HAVING filters after GROUP BY.

Advanced Aggregation Patterns

Use COUNT DISTINCT, rollups, and grouping sets for richer summaries.

🧒 ELI5

Count unique toys

You count how many different types of toys you have.

🔗 ANALOGY

Sales by region and total

You want totals per region and overall.

Technical

What it means in real code

Definition: GROUPING SETS and ROLLUP compute multiple group levels in one query.

Why it matters

  • Efficient analytics
  • Fewer queries
  • Consistent totals

Common pitfall

  • Multiple scans for totals
  • Manual unions
  • Incorrect totals
COUNT DISTINCTROLLUPGROUPING SETS
Group by key→Add rollup→Compute totals→Return results

Interview answer

“I use grouping sets to calculate per-dimension totals in one query.”

FeatureUse
ROLLUPHierarchical totals
CUBEAll combinations
ROLLUP example
SELECT region, product, SUM(amount)
FROM sales
GROUP BY ROLLUP(region, product);
Warning

Grouping sets can be large

ROLLUP/CUBE can generate many rows.

Mini takeaway
  • Use rollups for totals.
  • Avoid extra queries.
  • Watch output size.

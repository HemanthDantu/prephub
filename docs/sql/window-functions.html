Window Basics

Window functions compute values across related rows without collapsing them.

🧒 ELI5

Compare with neighbors

You can see how tall you are compared to classmates.

🔗 ANALOGY

Race standings

Every runner keeps their time, plus their rank.

Technical

What it means in real code

Definition: Window functions use OVER(PARTITION BY ... ORDER BY ...) to compute analytics per group.

Why it matters

  • Keeps row-level detail
  • Powerful analytics
  • Single-pass calculations

Common pitfall

  • GROUP BY misuse
  • Multiple subqueries
  • Hard-to-read SQL
OVERPARTITIONORDER
Define partition→Order rows→Compute window→Return results

Interview answer

“Window functions compute analytics per group without losing detail.”

FunctionUse
ROW_NUMBERRow index
RANKRanking
Row number
SELECT id, amount,
  ROW_NUMBER() OVER (PARTITION BY customer_id ORDER BY amount DESC) AS rn
FROM orders;
Warning

Large partitions cost more

Window functions process partitions; big groups can be expensive.

Mini takeaway
  • Use window functions for analytics.
  • Partition carefully.
  • Keep queries readable.
Quiz

What does PARTITION BY do?

Correct answer: B Partition defines the window group.
Section 2 of 4 · ~7 mins

Ranking Functions

RANK, DENSE_RANK, and ROW_NUMBER solve top-N per group problems.

🧒 ELI5

Gold, silver, bronze

You can rank players even if ties happen.

🔗 ANALOGY

Leaderboard

Ranks show ordering; ties may skip numbers.

Technical

What it means in real code

Definition: ROW_NUMBER gives a unique order; RANK allows gaps; DENSE_RANK removes gaps.

Why it matters

  • Top-N queries
  • Leaderboard logic
  • Tie handling

Common pitfall

  • Wrong rank for ties
  • Missing ORDER BY
  • Group-by misuse
ROW_NUMBERRANKDENSE_RANK
Order rows→Assign rank→Filter by rank→Return top-N

Interview answer

“I use ROW_NUMBER for unique order and DENSE_RANK when ties should share rank.”

FunctionTie handling
ROW_NUMBERNo ties
RANKGaps
DENSE_RANKNo gaps
Top 3 per group
SELECT * FROM (
  SELECT *, ROW_NUMBER() OVER (PARTITION BY dept ORDER BY salary DESC) rn
  FROM employees
) t WHERE rn <= 3;
Warning

Always define ORDER BY

Without ORDER BY, ranking is meaningless.

Mini takeaway
  • Pick correct rank function.
  • Always ORDER BY.
  • Filter using outer query.
Quiz

Which function removes gaps in ranking?

Correct answer: B DENSE_RANK avoids gaps in ties.
Section 3 of 4 · ~7 mins

Running Totals and Offsets

Use SUM OVER for running totals and LAG/LEAD for comparisons.

🧒 ELI5

Keep a running score

You add points as the game continues.

🔗 ANALOGY

Bank balance

Balance changes with each transaction.

Technical

What it means in real code

Definition: Window frames define range for aggregates; LAG/LEAD access nearby rows.

Why it matters

  • Trend analysis
  • Time series comparisons
  • Analytics

Common pitfall

  • Wrong frame
  • Unbounded ranges
  • Slow queries
FramesLAGLEAD
Order rows→Define frame→Compute aggregate→Compare offsets

Interview answer

“Running totals use SUM OVER with ORDER BY and a frame.”

FunctionUse
SUM OVERRunning total
LAG/LEADCompare rows
Running total
SELECT day, amount,
  SUM(amount) OVER (ORDER BY day) AS running_total
FROM sales;
Warning

Frames change meaning

Use ROWS BETWEEN to control window size.

Mini takeaway
  • Define correct frames.
  • Use LAG/LEAD for comparison.
  • Avoid unbounded ranges when needed.
Quiz

Which function accesses the previous row?

Correct answer: B LAG returns the previous row’s value.

Window functions can be heavy; indexes and partition sizes matter.

Big class is harder

🔗 ANALOGY

Huge ledger

Summing a big ledger takes longer.

Technical

What it means in real code

Definition: Sorting and partitioning drive cost; indexes help ORDER BY operations.

Why it matters

  • Predictable performance
  • Scalable analytics
  • Avoids timeouts

Common pitfall

  • Large sorts
  • No indexes
  • Huge partitions

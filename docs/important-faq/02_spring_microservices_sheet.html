Spring & Microservices Sheet

Spring Boot, REST, dependency injection, service boundaries, and the architecture choices that matter in production.

What to focus on

This sheet helps you connect Spring Boot internals with real service design decisions: DI, auto-configuration, web layers, persistence, security, and testing.

  • Explain why Spring uses proxies and how that changes transaction or AOP behavior.
  • Connect REST design with service communication and failure handling.
  • Review how JPA, transactions, and security fit into the runtime model.

Study cue

Use this after Java Core and before the AWS and backend reference sheets.

Section 1 of 4 · ~7 mins

Images vs Containers

Images are immutable templates; containers are running instances.

🧒 ELI5

Recipe vs meal

A recipe is the plan, the meal is the result.

🔗 ANALOGY

Blueprint vs building

A blueprint becomes a building when constructed.

Technical

What it means in real code

Definition: Docker images are layered filesystems; containers add a writable layer at runtime.

Why it matters

  • Portable runtime
  • Consistent environments
  • Fast deployments

Common pitfall

  • Mutating containers
  • No versioned images
  • Environment drift
ImageContainerLayer
Build image→Run container→Expose ports→Stop/replace

Interview answer

“Images are immutable; containers are disposable. Rebuild rather than patch.”

ArtifactMeaning
ImageTemplate
ContainerRunning instance
Build and run
docker build -t app:1.0 .
docker run -p 8080:8080 app:1.0
Warning

Do not treat containers like pets

They should be disposable and reproducible.

Mini takeaway
  • Images are immutable.
  • Containers are ephemeral.
  • Rebuild often.
Quiz

What is a container?

Correct answer: B A container is a running instance of an image.
Section 2 of 4 · ~7 mins

Dockerfile and Layers

Dockerfiles define how images are built. Each line creates a cached layer.

🧒 ELI5

Stack of pancakes

Each pancake is a layer; you can reuse layers for speed.

🔗 ANALOGY

Assembly line steps

Each step adds to the final product.

Technical

What it means in real code

Definition: Each Dockerfile instruction creates a layer; caching speeds rebuilds.

Why it matters

  • Faster builds
  • Smaller images
  • Deterministic releases

Common pitfall

  • Large base images
  • Copying everything early
  • No .dockerignore
LayersCacheSlim base
Choose base→Copy files→Install deps→Set CMD

Interview answer

“I order Dockerfile steps to maximize cache and reduce image size.”

TipBenefit
Use slim baseSmaller image
Copy deps firstCache reuse
Optimized Dockerfile
FROM eclipse-temurin:21-jre
WORKDIR /app
COPY build/libs/app.jar app.jar
EXPOSE 8080
CMD ["java","-jar","app.jar"]
Warning

Large images slow deployments

Big images increase transfer time and cold starts.

Mini takeaway
  • Order steps for cache.
  • Use slim base images.
  • Add .dockerignore.
Quiz

Why copy dependency files first?

Correct answer: B It allows dependency install layers to be cached.
Section 3 of 4 · ~7 mins

Volumes and Networking

Volumes persist data; networks connect containers securely.

🧒 ELI5

Toy box storage

You keep toys in a box so they’re still there tomorrow.

🔗 ANALOGY

Apartment plumbing

Pipes connect apartments; storage keeps water available.

Technical

What it means in real code

Definition: Volumes mount persistent storage; networks create isolated communication channels.

Why it matters

  • Data persistence
  • Service isolation
  • Secure connectivity

Common pitfall

  • Storing data in container FS
  • Using host network
  • Unsecured ports
VolumesBridge networkIsolation
Create volume→Mount volume→Attach network→Communicate

Interview answer

“I use volumes for data and user-defined networks for clean service discovery.”

FeatureUse
VolumePersistent data
NetworkService discovery
Volume mount
docker run -v data:/var/lib/postgresql/data postgres:16
Warning

Data loss without volumes

Container files are deleted when the container is removed.

Mini takeaway
  • Persist data with volumes.
  • Use custom networks.
  • Avoid host networking.
Quiz

Why use volumes?

Correct answer: B Volumes persist data beyond container lifetime.
Section 4 of 4 · ~7 mins

Docker Best Practices

Security and reliability depend on good defaults and minimal images.

🧒 ELI5

Wear a helmet

Safety rules keep you protected.

🔗 ANALOGY

Seat belts

Small habits prevent big problems.

Technical

What it means in real code

Definition: Use non-root users, health checks, and minimal images to improve security.

Why it matters

  • Lower attack surface
  • Safer deployments
  • Better observability

Common pitfall

  • Running as root
  • No health checks
  • Secret in image
Non-rootHealthcheckSecrets
Create user→Add health check→Scan image→Deploy

Interview answer

“I run containers as non-root and add health checks for orchestration.”

PracticeBenefit
Non-rootSecurity
HealthcheckReliability
Healthcheck
HEALTHCHECK --interval=30s --timeout=3s   CMD curl -f http://localhost:8080/actuator/health || exit 1
Warning

Do not bake secrets into images

Use env vars or secret managers.

Mini takeaway
  • Run as non-root.
  • Add health checks.
  • Keep images small.
Quiz

Why avoid running as root?

Correct answer: C Root privileges increase attack surface.

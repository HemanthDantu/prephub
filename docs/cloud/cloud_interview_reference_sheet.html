AWS Availability & Regions

2 questions

Answer: An Availability Zone is a physically separate data center inside an AWS Region. Each AZ has independent power, cooling, and networking, but low-latency connectivity to other AZs in the same region.

Why it matters: It improves fault tolerance and high availability by reducing the risk of a single data center failure taking down the whole application.

Advantage: Services can be deployed across multiple AZs to avoid single points of failure.

Answer: A Region is a geographic area such as us-east-1. An Availability Zone is an isolated data center within that region such as us-east-1a.

Why it matters: Regions help with latency, geography, and data residency. AZs help with high availability inside a region.

Advantage: You can choose a region close to users and still deploy redundantly across multiple AZs within that region.

Messaging & Eventing

3 questions

Answer: SQS itself does not do message filtering. Common patterns are:

  • Attach message attributes and let consumers decide whether to process or ignore them.
  • Use SNS in front of SQS and apply SNS filter policies to route events to different queues.
  • Create separate queues for different message types.

Why it matters: It keeps producers and consumers decoupled and makes routing explicit.

Advantage: Cleaner service ownership, easier scaling, and better separation of workloads.

SQS: Queue-based async messaging. One consumer processes a given message.

SNS: Pub/sub fan-out. One message can be delivered to many subscribers.

EventBridge: Event bus for routing events across AWS services and custom apps using rules.

Why it matters: They solve different async communication patterns.

Advantage: Decoupled services, better scalability, and more resilient integrations.

Answer: Both are messaging systems. Google Cloud Pub/Sub is fully managed and simpler to operate. Kafka gives more control over partitions, retention, ordering, and stream processing, but requires more operational ownership unless using a managed Kafka service.

Why it matters: The choice depends on whether you want ease of use or deeper control.

Advantage: Pub/Sub reduces ops overhead. Kafka supports richer streaming use cases and ecosystem tooling.

Lambda & Serverless

9 questions

Answer: Write the function code, choose a runtime, configure handler, memory, timeout, and permissions, then deploy using the AWS Console, AWS CLI, or infrastructure as code such as Terraform.

Why it matters: Lambda is event-driven compute without server management.

Advantage: Auto-scaling, pay-per-use, and reduced ops overhead.

Answer: Create an IAM role with the required S3 permissions and attach that role to the Lambda function.

Why it matters: Lambda should access AWS resources securely without hard-coded credentials.

Advantage: Safer access control and support for least-privilege design.

Answer: In Java, create a Maven project, add the AWS Lambda dependencies, implement a handler such as RequestHandler, and package the project as a JAR with Maven.

public class Handler implements RequestHandler<Map<String, String>, String> {
    public String handleRequest(Map<String, String> input, Context context) {
        return "Hello Lambda";
    }
}

Why it matters: Maven gives dependency management and a repeatable build process for Java Lambdas.

Advantage: Clean packaging and easy CI/CD integration.

Answer: Define the Lambda function, execution role, handler, runtime, memory, timeout, environment variables, and code source in Terraform. Then run terraform apply to create or update it.

Why it matters: It keeps infrastructure versioned and repeatable.

Advantage: Consistent deployments, reviewable changes, and easier environment management.

Answer: Package the code as a zip or JAR and update the function using the AWS CLI.

aws lambda update-function-code \
  --function-name my-fn \
  --zip-file fileb://function.zip

Why it matters: It enables fast scripted deployments.

Advantage: Easy automation in CI/CD pipelines.

AWS Networking & Service Communication

7 questions

Answer: Services typically communicate through synchronous REST APIs or asynchronous messaging systems such as SQS, SNS, EventBridge, or streaming platforms like Kafka/MSK.

Why it matters: The communication pattern affects coupling, latency, reliability, and scaling.

Advantage: Async messaging improves resilience and decoupling. APIs provide immediate responses when needed.

Answer: API Gateway is a managed service that exposes HTTP or REST APIs and routes requests to backends such as Lambda, EC2, or other services.

Why it matters: It acts as the controlled entry point to backend systems.

Advantage: Built-in auth, throttling, monitoring, and scaling.

Answer: IAM is AWS Identity and Access Management. It defines who can access which AWS resources and what actions they can perform, using users, roles, and policies.

Why it matters: Every AWS architecture needs secure access control.

Advantage: Fine-grained permissions and least-privilege security.

Answer: A VPC is a logically isolated private network in AWS where you define IP ranges, subnets, routing, and security boundaries for your resources.

Why it matters: It provides controlled networking for cloud systems.

Advantage: Better isolation, security, and network design flexibility.

Answer: A Load Balancer distributes incoming traffic across multiple backend instances and performs health checks so traffic avoids unhealthy targets.

Why it matters: It prevents one instance from becoming a bottleneck and supports high availability.

Advantage: Better scalability, fault tolerance, and traffic management.

Compute Engine: Virtual machines where you manage the environment yourself.

App Engine: Platform as a service where Google manages more of the infrastructure for you.

Why it matters: It is the same control vs convenience decision seen in other cloud platforms.

Advantage: App Engine speeds up delivery. Compute Engine gives more customization.

ALB: Application Load Balancer works at Layer 7, understands HTTP/HTTPS, and supports host-based and path-based routing.

NLB: Network Load Balancer works at Layer 4, handles TCP/UDP, and is used when you need very high performance or non-HTTP traffic.

Why it matters: The protocol and routing needs determine which load balancer fits.

Advantage: ALB is ideal for web apps and microservices. NLB is ideal for lower-level network traffic and very high throughput.

Cloud Fundamentals

4 questions

Answer: Cloud computing is the on-demand delivery of compute, storage, and networking over the internet with pay-as-you-go pricing.

Why it matters: Teams do not need to buy and manage physical hardware upfront.

Advantage: Faster provisioning, elasticity, and reduced infrastructure management.

IaaS: Infrastructure as a service. You manage the app and much of the runtime environment. Example: EC2 or Compute Engine.

PaaS: Platform as a service. The provider manages more of the platform. Example: App Engine or Elastic Beanstalk.

SaaS: Software as a service. You consume the software directly. Example: Gmail or Salesforce.

Why it matters: It defines how much operational responsibility your team takes on.

Advantage: Lets teams balance flexibility with simplicity.

Answer: Auto scaling automatically adds or removes compute capacity based on demand or metrics such as CPU or request volume.

Why it matters: Traffic changes over time and systems should adapt automatically.

Advantage: Better availability during spikes and lower cost during low traffic periods.

Vertical scaling: Add more CPU or memory to a single machine.

Horizontal scaling: Add more machines and distribute traffic across them.

Why it matters: Cloud-native systems usually favor horizontal scaling because it supports resilience and growth better.

Advantage: Horizontal scaling improves fault tolerance and avoids single-machine limits.

3 questions

Answer: Docker packages an application and its dependencies into a container so it runs consistently across environments.

Advantage: Consistency, portability, and faster deployment workflows.

Why it matters: Different systems need different tradeoffs between recovery speed and expense.

Advantage: Better business continuity planning and reduced outage impact.

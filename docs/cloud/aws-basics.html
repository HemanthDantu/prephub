IAM Fundamentals

IAM is the control plane for who can do what in AWS via users, roles, and policies.

🧒 ELI5

Keys and doors

You give each person the right key for the right door.

🔗 ANALOGY

Office badges

Badges control which rooms you can enter.

Technical

What it means in real code

Definition: IAM policies define permissions; roles are assumed identities; users are human accounts.

Why it matters

  • Least privilege
  • Auditability
  • Cross-account access

Common pitfall

  • Using root account
  • Wildcard policies
  • Long-lived keys
RolesPoliciesLeast privilege
Authenticate→Assume role→Evaluate policy→Grant/deny

Interview answer

“IAM controls access with policies and roles; use least privilege and short-lived creds.”

IdentityUse
UserHuman login
RoleAssumed identity
Policy snippet
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": ["s3:GetObject"],
      "Resource": "arn:aws:s3:::my-bucket/*"
    }
  ]
}
Warning

Avoid wildcard permissions

Use specific actions/resources to reduce blast radius.

Mini takeaway
  • Use roles for apps.
  • Rotate keys.
  • Scope policies.
Quiz

What is the recommended identity for applications on EC2?

Correct answer: B Use IAM roles attached to instances for short-lived credentials.

EC2 Compute Basics

EC2 provides virtual machines with instance types, storage, and autoscaling.

🧒 ELI5

Rent a computer

You rent a computer that runs all day and pay for the time.

🔗 ANALOGY

Apartment leasing

Choose a size and pay for what you need.

Technical

What it means in real code

Definition: EC2 instances are virtual servers with AMIs, security groups, and EBS volumes.

Why it matters

  • Flexible compute
  • Autoscaling support
  • Custom networking

Common pitfall

  • Always-on instances
  • No autoscaling
  • Weak security groups
AMIEBSASG
Select AMI→Choose instance type→Attach storage→Launch + secure

Interview answer

“EC2 is virtual compute; combine with autoscaling and security groups for resilience.”

ComponentRole

Overprovisioning

Right-size instances and use autoscaling to avoid wasted cost.

Mini takeaway
  • Use ASGs.
  • Lock down security groups.
  • Tag resources.
Quiz

What provides a virtual firewall for EC2 instances?

Correct answer: B Security groups control inbound/outbound traffic at instance level.
Section 3 of 5 · ~7 mins

S3 Storage Essentials

S3 is durable object storage with buckets, lifecycle rules, encryption, and access policies.

🧒 ELI5

Toy storage

You keep toys in labeled boxes and decide when to move them to the attic.

🔗 ANALOGY

Warehouse shelves

Store items by box and track how long they stay.

Technical

What it means in real code

Definition: S3 stores objects in buckets, secured with bucket policies, ACLs, and encryption.

Why it matters

  • 11 9s durability
  • Cheap storage
  • Event-driven triggers

Common pitfall

  • Public buckets
  • No lifecycle
  • No encryption
BucketsLifecycleEncryption
Put object→Store in bucket→Apply lifecycle→Serve or archive

Interview answer

“S3 is object storage; secure with bucket policies and lifecycle rules.”

Storage classUse
StandardFrequent access
GlacierArchive
Lifecycle rule
{
  "Rules": [
    {
      "ID": "archive",
      "Status": "Enabled",
      "Transitions": [{"Days": 30, "StorageClass": "GLACIER"}]
    }
  ]
}
Warning

Public access leaks

Use block public access unless you intentionally host static content.

Mini takeaway
  • Use bucket policies.
  • Set lifecycle transitions.
  • Encrypt by default.
Quiz

What does S3 store?

Correct answer: B S3 is object storage; data is stored as objects in buckets.
Section 4 of 5 · ~6 mins

VPC & Networking Basics

A VPC is your private network in AWS with subnets, route tables, gateways, and security layers.

🧒 ELI5

Neighborhood map

You decide which houses are public and which are private.

🔗 ANALOGY

Office building

Some floors are public, others need a badge.

Technical

What it means in real code

Definition: VPC isolates resources with subnets, route tables, NAT/IGWs, and security groups/NACLs.

Why it matters

  • Network isolation
  • Control ingress/egress
  • Multi-tier architecture

Common pitfall

  • Everything in public subnet
  • No NAT for private instances
  • Open NACLs
SubnetsRoute tablesNAT
Create VPC→Add subnets→Attach gateways→Apply security

Interview answer

“A VPC is isolated networking; use public/private subnets and secure with SGs and NACLs.”

ComponentPurpose
IGWPublic internet
NATPrivate outbound
Security group idea
Inbound: 443 from 0.0.0.0/0
Outbound: 0.0.0.0/0
Warning

Public subnets for everything

Keep databases in private subnets with no direct internet access.

Mini takeaway
  • Use public/private split.
  • Control routes.
  • Avoid 0.0.0.0/0 for sensitive ports.
Quiz

Which component provides outbound internet for private subnets?

Correct answer: B NAT gateways allow private subnets to access the internet outbound.

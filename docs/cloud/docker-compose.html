Section 2 of 4 · ~7 mins

Service Dependencies and Health

depends_on controls startup order; health checks ensure readiness.

🧒 ELI5

Wait for the oven

You don’t bake before the oven is hot.

🔗 ANALOGY

Stage setup

You wait until lights are on before performers go out.

Technical

What it means in real code

Definition: depends_on orders startup but does not guarantee service readiness unless health checks are used.

Why it matters

  • Avoids startup failures
  • Improves reliability
  • Clear dependencies

Common pitfall

  • App fails on startup
  • Race conditions
  • No health checks
depends_onhealthcheckreadiness
Start db→Wait health→Start api→Connect

Interview answer

“I add health checks so depends_on waits for real readiness.”

FeatureNote
depends_onOrder only
healthcheckReadiness
Health check
services:
  db:
    image: postgres:16
    healthcheck:
      test: ["CMD", "pg_isready", "-U", "postgres"]
Warning

Order != readiness

depends_on does not wait for readiness by default.

Mini takeaway
  • Add health checks.
  • Avoid race conditions.
  • Document dependencies.
Quiz

What does depends_on guarantee?

Correct answer: B depends_on only guarantees order, not readiness.

Networks and Volumes

Compose creates networks for service discovery and volumes for persistence.

🧒 ELI5

Shared hallway

Rooms connect through a shared hallway.

🔗 ANALOGY

Shared storage closet

You keep common items in a shared closet.

Technical

What it means in real code

Definition: Compose networks enable DNS discovery by service name; volumes persist data.

Why it matters

  • Stable connectivity
  • Persistent databases
  • Simpler dev

Common pitfall

  • Hard-coded IPs
  • Data loss
  • Mixed networks
NetworkVolumeDNS
Create network→Attach services→Resolve by name→Persist data

Interview answer

“I use service names for DNS and volumes for data persistence.”

ElementPurpose
NetworkDiscovery
VolumeState
Named volume
volumes:
  dbdata:
services:
  db:
    volumes:
      - dbdata:/var/lib/postgresql/data
Warning

No volume means data loss

Containers remove data when deleted.

Mini takeaway
  • Use networks for discovery.
  • Use volumes for state.
  • Avoid hard-coded IPs.
Quiz

Correct answer: B Compose provides DNS by service name on the network.

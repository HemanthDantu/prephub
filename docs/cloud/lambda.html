Function Model & Runtime

Lambda runs your handler in a managed runtime with configurable memory and timeout.

🧒 ELI5

Instant helper

A helper appears to do one task, then leaves.

🔗 ANALOGY

Food truck

The kitchen opens when customers arrive, then closes.

Technical

What it means in real code

Definition: Lambda executes code in a short-lived environment, scaling by creating more instances.

Why it matters

  • No server management
  • Pay per use
  • Automatic scaling

Common pitfall

  • Long-running tasks
  • Stateful logic
  • Ignoring timeouts
handlertimeoutmemory
Event arrives→Runtime loads→Handler executes→Returns response

Interview answer

“Lambda runs a handler per event in an ephemeral container with memory/timeout limits.”

SettingImpact
MemoryCPU allocation
TimeoutMax runtime
Node.js handler
exports.handler = async (event) => {
  return { statusCode: 200, body: "ok" };
};
Warning

Not for long jobs

If you need hours-long processing, use ECS or Batch.

Mini takeaway
  • Keep functions small.
  • Tune memory/timeouts.
  • Use stateless design.
Quiz

Lambda pricing is primarily based on?

Correct answer: B Pricing is based on request count and execution duration.

Section 3 of 5 · ~6 mins

Cold Starts & Performance

Cold starts add latency when a new environment is initialized.

🧒 ELI5

Warming up the oven

The first cookie takes longer because the oven is cold.

🔗 ANALOGY

Starting a car

The first start takes longer on a cold morning.

Technical

What it means in real code

Definition: Cold starts happen when AWS provisions a new execution environment; warm starts reuse it.

Why it matters

  • Affects latency
  • Impacts user experience
  • Tunable via memory and provisioned concurrency

Common pitfall

  • Large dependencies
  • VPC cold start penalty
  • No performance testing
cold startprovisionedmemory
Provision env→Init runtime→Load code→Handle event

Interview answer

“Cold starts add latency; reduce by trimming dependencies or using provisioned concurrency.”

TacticEffect
Provisioned concurrencyWarm containers
Smaller packageFaster init
Provisioned concurrency (CLI)
aws lambda put-provisioned-concurrency-config   --function-name apiHandler   --qualifier live   --provisioned-concurrent-executions 5
Warning

Costs for provisioned concurrency

Provisioned concurrency has a cost even when idle.

Mini takeaway
  • Measure cold start latency.
  • Reduce package size.
  • Use provisioned concurrency if needed.
Quiz

What primarily causes a cold start?

Correct answer: B Cold starts occur when AWS initializes a new execution environment.

Security & IAM Roles

Lambda executes with an IAM role; permissions should be minimal and scoped.

🧒 ELI5

Limited toolbox

Give the helper only the tools needed for the job.

🔗 ANALOGY

Temporary badge

A contractor gets a badge that only opens a few doors.

Technical

What it means in real code

Definition: Execution role defines what AWS resources Lambda can access; environment variables store config.

Why it matters

  • Least privilege
  • Auditability
  • Reduced blast radius

Common pitfall

  • * permissions
  • Hardcoded credentials
  • No secrets manager
IAM roleSecretsLeast privilege
Assign role→Evaluate policies→Allow/deny request→Log to CloudTrail

Interview answer

“Lambda uses an execution role; keep policies narrow and store secrets securely.”

ApproachBest
Secrets ManagerManaged secrets
Env varsNon-sensitive config
Policy for S3 read

  "Effect": "Allow",

Warning

Overly broad roles

Avoid granting full access to S3 or DynamoDB.

Mini takeaway
  • Use least privilege.
  • Rotate secrets.
  • Audit with CloudTrail.

What should hold database passwords?

Section 5 of 5 · ~6 mins

Versioning, Aliases & Layers

Versions make deployments safe; aliases route traffic; layers share dependencies.

🧒 ELI5

Save game slots

You can go back to a previous save if needed.

🔗 ANALOGY

Traffic switchboard

You route calls to the right line.

Technical

What it means in real code

Definition: Lambda versions are immutable snapshots; aliases are pointers; layers package shared code.

Why it matters

  • Safe deploys
  • Blue/green releases
  • Reusable dependencies

Common pitfall

  • No versioning
  • Large layers
  • Aliases not updated
versionsaliaseslayers
Publish version→Update alias→Shift traffic→Roll back if needed

Interview answer

“Use versions for immutable releases, aliases for routing, and layers for shared libraries.”

ItemRole
AliasTraffic pointer
LayerShared deps
Alias update (CLI)
aws lambda update-alias   --function-name apiHandler   --name live   --function-version 12
Warning

No version pinning

If you use $LATEST everywhere, rollbacks are risky.

Mini takeaway
  • Publish versions.
  • Use aliases for routing.
  • Keep layers small.
Quiz

What is the purpose of a Lambda alias?

Correct answer: B Aliases point to a specific version and can shift traffic.

Cluster and Nodes

Kubernetes clusters coordinate nodes and schedules workloads.

🧒 ELI5

Team manager

A manager assigns tasks to team members.

🔗 ANALOGY

Air traffic control

Workloads are routed to the best runway.

Technical

What it means in real code

Definition: The control plane schedules pods on worker nodes and manages desired state.

Why it matters

  • Automated scheduling
  • Self-healing
  • Scaling

Common pitfall

  • Manual restarts
  • Single-node failures
  • No autoscaling
Control planeNodesScheduler
Define desired state→Scheduler assigns node→Kubelet runs pod→Controller reconciles

Interview answer

“K8s control plane enforces desired state by scheduling pods on nodes.”

ComponentRole
API ServerControl plane entry
SchedulerAssign pods
Check nodes
kubectl get nodes
Warning

No limits = noisy neighbors

Pods without limits can starve others.

Mini takeaway
  • Control plane manages state.
  • Scheduler places pods.
  • Always set limits.
Quiz

What schedules pods to nodes?

Correct answer: B The scheduler assigns pods to nodes.
Section 2 of 4 · ~7 mins

Pods and Deployments

Pods run containers; Deployments manage replica sets and updates.

🧒 ELI5

Basket of eggs

A pod holds a few eggs together.

🔗 ANALOGY

Orchestra section

Deployments keep a fixed number of players.

Technical

What it means in real code

Definition: Deployments manage ReplicaSets to keep pods running and updated.

Why it matters

  • Self-healing
  • Rolling updates
  • Scaling

Common pitfall

  • Manually created pods
  • No readiness probes
  • Single replica
PodDeploymentReplicaSet
Define deployment→Create replica set→Schedule pods→Roll updates

Interview answer

“Deployments manage replicas and rolling updates via ReplicaSets.”

ObjectUse
PodRun containers
DeploymentManage replicas
Deployment snippet
apiVersion: apps/v1
kind: Deployment
metadata:
  name: api
spec:
  replicas: 3
  template:
    spec:
      containers:
      - name: api
        image: api:1.0
Warning

No readiness = bad rollouts

Without probes, traffic may hit broken pods.

Mini takeaway
  • Use deployments for updates.
  • Configure probes.
  • Scale replicas.
Quiz

What object manages rolling updates?

Correct answer: B Deployments manage rolling updates.
Section 3 of 4 · ~7 mins

Services and Ingress

Services provide stable endpoints; Ingress routes external traffic.

🧒 ELI5

Front desk

You call the front desk instead of each room.

🔗 ANALOGY

Restaurant host

The host routes guests to tables.

Technical

What it means in real code

Definition: Services provide stable DNS/IP; Ingress routes HTTP traffic from outside.

Why it matters

  • Stable networking
  • Load balancing
  • External access

Common pitfall

  • Direct pod IPs
  • Missing selectors
  • Ingress without TLS
ServiceIngressTLS
Create service→Select pods→Expose ingress→Route traffic

Interview answer

“Services give stable endpoints; Ingress handles external routing.”

TypeUse
ClusterIPInternal
LoadBalancerExternal
Service snippet
apiVersion: v1
kind: Service
metadata:
  name: api
spec:
  selector:
    app: api
  ports:
  - port: 80
    targetPort: 8080
Warning

Ingress without TLS is risky

Always secure external traffic.

Mini takeaway
  • Use services for stable endpoints.
  • Ingress routes external traffic.
  • Enable TLS.
Quiz

What gives a stable DNS name in K8s?

Correct answer: B Services provide stable networking.
Section 4 of 4 · ~7 mins

ConfigMaps and Secrets

ConfigMaps and Secrets externalize configuration.

🧒 ELI5

Recipe notes

You keep notes outside the recipe book.

🔗 ANALOGY

Locked drawer

Secrets go in a locked drawer.

Technical

What it means in real code

Definition: ConfigMaps store non-sensitive config; Secrets store sensitive data.

Why it matters

  • Decouple config
  • Secure secrets
  • Simpler updates

Common pitfall

  • Secrets in YAML
  • Hard-coded config
  • No rotation
ConfigMapSecretRotation
Create config→Mount into pod→Reload app→Rotate secrets

Interview answer

“I use ConfigMaps for config and Secrets for sensitive values.”

ResourceUse
ConfigMapNon-sensitive
SecretSensitive
ConfigMap example
kubectl create configmap app-config --from-literal=MODE=prod
Warning

Secrets are base64, not encrypted

Use KMS or sealed secrets for stronger security.

Mini takeaway
  • Externalize config.
  • Keep secrets safe.
  • Plan rotation.
Quiz

What stores sensitive data in K8s?

Correct answer: B Secrets store sensitive configuration.

Simple @Around advice
@Aspect
@Component
public class TimingAspect {
  @Around("execution(* com.acme..*(..))")
  public Object time(ProceedingJoinPoint pjp) throws Throwable {
    long start = System.currentTimeMillis();
    try { return pjp.proceed(); }
    finally { log.info("{} took {}ms", pjp.getSignature(), System.currentTimeMillis() - start); }
  }
}

Proxy Types: JDK vs CGLIB

Spring can proxy interfaces with JDK dynamic proxies or classes with CGLIB.

🧒 ELI5

Stand-in actor

A stand-in actor can play a role as long as they follow the same script.

🔗 ANALOGY

Adapter vs mold

An adapter fits a standard plug, while a mold copies the shape exactly.

Technical

What it means in real code

Definition: JDK proxies implement interfaces; CGLIB creates subclasses to proxy concrete classes.

Why it matters

  • Works with or without interfaces
  • Flexible proxying
  • Control over behavior

Common pitfall

  • No interfaces with JDK proxies
  • Final classes with CGLIB
  • Unexpected proxy type

Final methods are not advised

CGLIB cannot override final methods, so advice will not apply.

Mini takeaway
  • JDK proxies need interfaces.
  • CGLIB proxies classes.
  • Final methods cannot be advised.

Self-invocation and AOP Limitations

Calls within the same class bypass the proxy, so advice does not apply.

🧒 ELI5

Calling yourself

If you call yourself, the assistant does not get a chance to help.

🔗 ANALOGY

Taking a shortcut

If you bypass the checkpoint, no one checks your pass.

Technical

What it means in real code

Definition: Spring AOP works through proxies, so only external calls go through the proxy and trigger advice.

Why it matters

  • Prevents false assumptions
  • Explains missing advice
  • Guides design choices

Common pitfall

  • Advice not applied
  • Calling methods internally
  • Unexpected transaction boundaries

Edge cases

Edge Cases & Tradeoffs

  • Self-invocation bypasses proxies.
  • Final classes/methods cannot be proxied by CGLIB.
  • Proxying changes equals/hashCode semantics.
  • Ordering of aspects can change behavior.
  • Advice with heavy logic slows calls.
Checklist

Production Checklist

  • Keep aspects small and focused.
  • Document ordering with @Order.
  • Avoid AOP for core business logic.
  • Use interface-based proxies when possible.
  • Test cross-cutting behavior separately.

Actuator Endpoints and Health Checks

Actuator gives you standardized endpoints for health, info, metrics, and runtime diagnostics.

← Back to Spring Boot
🧒 ELI5

Dashboard lights

Like a car dashboard that tells you if the engine is healthy or needs attention.

🔗 ANALOGY

Factory control panel

Operators check the panel to see if each system is running.

Technical

What it means in real code

Definition: Spring Boot Actuator adds production-ready endpoints under /actuator with configurable exposure.

Why it matters

  • Standardized health checks
  • Easy monitoring
  • Operational visibility

Common pitfall

  • Custom health endpoints everywhere
  • No readiness checks
  • Hidden failure signals
HealthInfoMetrics
Request /actuator/health→Run health indicators→Aggregate status→Return JSON

Interview answer

“Actuator gives built-in health, info, and metrics endpoints that operations can scrape or alert on.”

Expose health and metrics

  endpoints:

      exposure:

Warning

Do not expose everything

Actuator exposes sensitive details if you open all endpoints without security.

Mini takeaway
  • Use actuator for standard health checks.
  • Expose only what you need.
  • Secure the endpoints.

Section 2 of 4 · ~7 mins

Metrics with Micrometer

Micrometer is the metrics facade in Spring Boot. It records counters, timers, gauges, and tags.

🧒 ELI5

A fitness tracker

It counts your steps and heart rate so you can see how you are doing over time.

🔗 ANALOGY

Scoreboard stats

A scoreboard keeps a running tally of scores, misses, and time.

Technical

What it means in real code

Definition: Micrometer provides a vendor-neutral API to record application metrics and export them to backends.

Why it matters

  • Consistent metrics API
  • Tag-based slicing
  • Easy export to Prometheus or Datadog

Common pitfall

  • Custom metrics per service
  • No tags
  • Unbounded cardinality
CountersTimersTags
Record metric→Meter registry→Backend scrape→Dashboards/alerts

Interview answer

“Micrometer lets me record counters and timers once, then export to Prometheus, Datadog, or others.”

MetricUse
CounterTotal requests
TimerLatency distribution
GaugeCurrent queue depth
Custom metric example
@Component
public class CheckoutMetrics {
  private final Counter counter;
  public CheckoutMetrics(MeterRegistry registry) {
    this.counter = registry.counter("checkout.success");
  }
  public void markSuccess() { counter.increment(); }
}
Warning

High-cardinality tags

Do not tag metrics with user IDs or request IDs; it can explode time series and costs.

Mini takeaway
  • Micrometer is the metrics API.
  • Tags add dimensions, use carefully.
  • Export to a registry.
Quiz

Which Micrometer meter is best for latency tracking?

Correct answer: C Timers capture duration and distribution of latency.
Section 3 of 4 · ~7 mins

Tracing and Correlation

Tracing follows a request across services and logs by attaching trace and span IDs.

🧒 ELI5

Breadcrumb trail

You leave breadcrumbs so you can see the path you took through the forest.

🔗 ANALOGY

Package tracking

Every package scan updates the tracking history so you know where it has been.

Technical

What it means in real code

Definition: Tracing creates spans for each operation, linking them into a trace and exporting to systems like Zipkin or Tempo.

Why it matters

  • End-to-end visibility
  • Faster root-cause analysis
  • Correlated logs and metrics

Common pitfall

  • No trace IDs in logs
  • Hard to debug distributed calls
  • Manual correlation
Trace IDSpanPropagation
Incoming request→Start span→Propagate context→Export trace

Interview answer

“Tracing adds trace/span IDs to requests so I can follow a single request across services and logs.”

SignalRole
Trace IDRequest correlation
SpanOperation timing
BaggageExtra context
Enable tracing exporter
management:
  tracing:
    sampling:
      probability: 0.1
  otlp:
    tracing:
      endpoint: http://collector:4317
Warning

Sampling trade-offs

Sampling too low loses visibility; too high can be expensive. Tune based on traffic.

Mini takeaway
  • Tracing connects services.
  • Propagate context through headers.
  • Tune sampling rates.
Quiz

What does a trace ID do?

Correct answer: B Trace IDs connect all spans for a single request across services.
Section 4 of 4 · ~7 mins

Production Hardening

Observability requires sensible exposure, security, and readiness integration.

🧒 ELI5

Lock the control room

You let operators in, but not just anyone off the street.

🔗 ANALOGY

Museum backroom

The public sees exhibits, but the backroom is locked and monitored.

Technical

What it means in real code

Definition: Secure actuator endpoints, configure health groups, and integrate probes for safe deployments.

Why it matters

  • Protects sensitive data
  • Supports rolling deploys
  • Improves reliability

Common pitfall

  • Actuator open to the world
  • No readiness/liveness split
  • Unsecured metrics
SecurityProbesLeast exposure
Secure endpoints→Expose health group→K8s probes→Alerting

Interview answer

“I expose only health and metrics, secure everything else, and wire readiness/liveness for deploy safety.”

SettingGoal
health.probes.enabledK8s support
exposure.includeLimit endpoints
securityProtect diagnostics
Enable probes
management:
  health:
    probes:
      enabled: true
Warning

Security first

Never expose /actuator/env or /actuator/beans publicly without authentication.

Mini takeaway
  • Secure endpoints.
  • Expose only what you need.
  • Use probes for deploys.
Quiz

Why split liveness and readiness probes?

Correct answer: B Readiness indicates if the app should receive traffic; liveness indicates if it should restart.
